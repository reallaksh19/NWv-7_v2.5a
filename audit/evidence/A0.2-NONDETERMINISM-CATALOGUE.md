# A0 — Non-determinism source catalogue (Insight pipeline)

Snapshot under test: `public/newsdata/insight_2026-05-19.json`
(sha256 `ade1430dd96bb5f1`, contentHash `40f989d5da9c`, 877 stories).
Method: static enumeration by `grep` over `src/insight/src/**` (test files excluded),
cross-checked against the runtime N=10 harness (`A0.1-DET-01`).

## Class 1 — True randomness
| Source | Status |
|---|---|
| `Math.random()` anywhere in pipeline | **NONE** (grep clean) — good |
| Embeddings (`src/adapters/embeddingsAdapter.js`) | **Deterministic**: fixed 200-term vocab TF-IDF, no RNG, no network. Same corpus → same vectors. |

→ No true non-determinism. Any divergence must come from wall-clock or iteration order.

## Class 2 — Wall-clock coupling (the replay hazard)
| file:line | Use | Affects output? | Injection point? |
|---|---|---|---|
| `pipeline/pipeline.ts:129` | `ranAt: Date.now()` | Metadata only (run timestamp). Excluded from the A0 projection. | none |
| `pipeline/normalize.ts:150` | `computeFreshnessScore(publishedAt, referenceTime?)` → `refTime = referenceTime ?? Date.now()` | Yes — freshness component | **YES** (`referenceTime` param) |
| `pipeline/normalize.ts:242` | normalize fn `refTime = referenceTime ?? Date.now()` | Yes — story freshness at normalize | **YES** (`referenceTime` param) |
| `pipeline/temporalTier.ts:33` | `computeEventAnchor`: `windowStart = Date.now() - maxStoryAgeHours*…`; returns `max(windowStart, firstSeenAt)` | Yes — temporal tier → evolution roles, angle counts, balance gate | **NO** (no param; reads clock directly) |
| `cache/cacheManager.ts:34,101,148` | `ageMs = Date.now() - entry.fetchedAt` (TTL/staleness) | Yes — slot refetch decisions | none (TTL design) |
| `cache/cacheManager.ts:69` | `fetchedAt: Date.now()` on `setCachedSlot` | Yes — feeds the ageMs above | none |

**Replay consequence:** running a dated snapshot at a *different* wall-clock shifts
the temporal-tier anchor (temporalTier.ts:33 clamps to `now − window`). For the
2026-05-19 snapshot evaluated at 2026-06-22, every story sits far below the window,
collapsing temporal tiers. This is exactly why the benchmark (B3) needs a **virtual
clock**. Clock injection is **partially wired**: `normalize.ts` already accepts
`referenceTime`, but `temporalTier.ts` and `cacheManager.ts` read `Date.now()`
directly, so a clean virtual clock requires threading a clock through those two
modules (or a global `Date.now` shim, which the A0 harness uses to prove feasibility).

## Class 3 — Iteration / ordering
| file:line | Note |
|---|---|
| `cache/cacheManager.ts:16` | module-level `const store = new Map()` (F5-7 in-process cache). Persists across runs in one process → **must be cleared between replay runs**. `invalidateSlot` (`:79-81`) does `store.delete(slot)` (full removal); the A0 harness clears all 6 slots before each run. |
| `dedup/dedup.ts:933`, `cluster/topicCohesion.ts:76` | `localeCompare` used **only as a tie-breaker** when numeric scores are equal. Locale-stable for the ASCII angle/token labels in use, but technically locale-sensitive → flag if CI locale ever differs. |
| `ranking/ranking.ts:139,154,413`; `cluster/cluster.ts:92`; `tree/treeBuilder.ts:364,813`; `dedup/dedup.ts:509,931` | Numeric-key sorts (`b.x - a.x`). JS `Array.sort` is stable (Node ≥11), so equal keys preserve input order → deterministic given deterministic input order (which holds: fetcher returns snapshot order). Order-*sensitivity* (greedy seed order) is an A2.4 concern, not A0 non-determinism. |

## Class 4 — Environment
| Item | Note |
|---|---|
| Node version | CI/workflow uses Node 20 (`news_prefetch.yml`, `deploy.yml`); this audit ran Node `v22.22.2`. Cross-version byte-identity is **UNVERIFIED here** (no Node 20 in container) — flagged as a residual A0 item. |

## Bottom line
Determinism is achievable for a fixed (snapshot, clock) pair: no RNG, deterministic
embeddings, cache fully clearable. The sole divergence vector is wall-clock, and a
virtual clock is feasible (proven in `A0.1-DET-01` by pinning `Date.now`). Full
plumbing of an injected clock through `temporalTier.ts` + `cacheManager.ts` is the
remediation needed before the B3 replay benchmark can run on dated snapshots.
